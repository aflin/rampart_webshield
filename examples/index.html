rampart-webshield

Font-based text obfuscation for web pages

What is this?

This page demonstrates font-based anti-scraping protection. The text you are reading looks normal in your browser, but the page source contains scrambled Unicode characters. Bots and scrapers that read the raw HTML will only see meaningless character sequences.

Try it: right-click this page and select "View Page Source" to see the obfuscated HTML. Then try to copy and paste the text above. It will be garbled.

Module API

The rampart-webshield module exports a fontshuffle function that takes an HTML page, finds its font files, scrambles them, and rewrites the text. It can also scramble images using tile shuffling.

var ws = require("rampart-webshield");

// Simple mode (1-to-1 substitution):
var result = ws.fontshuffle("page.html", 42);

// Multi mode (many-to-1, defeats frequency analysis):
var result = ws.fontshuffle("page.html", 42, {mode: "multi"});

// Reuse saved mappings (skip font generation):
var result = ws.fontshuffle("page.html", 42, {mappings: saved});

// Enable image scrambling (requires rampart-gm):
var result = ws.fontshuffle("page.html", 42, {
    mode: "multi",
    images: true               // or {tileSize: 16} for custom
});

// Enable guard (CDP trap + delayed font loading):
var result = ws.fontshuffle("page.html", 42, {
    mode: "multi",
    guard: true            // or {delay: 300} for custom
});

// Result:
//   result.text     - obfuscated HTML string
//   result.fonts    - [{name: "font.ws.ttf", data: Buffer}, ...]
//   result.images   - [{name: "img.ws.jpg", data: Buffer}, ...]
//   result.mappings - mapping data (save as JSON to reuse)

// Scramble a single image directly:
var img = ws.scrambleImage("photo.jpg", 42, {tileSize: 32});
// img.image  = Buffer (scrambled image)
// img.width  = original width
// img.height = original height

Parameters

  • html - HTML text as a string, or a filename to read
  • seed - A positive number used to seed the PRNG
  • options.mode - "simple" (default) or "multi"
  • options.mappings - Previously saved mapping data to reuse
  • options.images - Enable image scrambling. Set to true for default 32px tiles, or {tileSize: N} for custom. All <img> tags are tile-shuffled and a client-side canvas script is injected to unscramble them. Requires rampart-gm module and the GraphicsMagick libraries (apt install libgraphicsmagick-q16-3).
  • options.guard - Enable CDP trap + delayed font loading. Set to true for default 500ms delay, or {delay: N} for custom. Automation frameworks like Puppeteer, Playwright, and Selenium are blocked from loading the font.

Font files are discovered automatically from @font-face declarations in inline styles and linked stylesheets. Both local files and remote URLs (fetched via rampart-curl) are supported.

No-scramble zones

Add data-no-scramble to any element to prevent its content from being scrambled. Use this for code blocks or other content that needs to be copy-pasteable. Pair it with a system font so the unscrambled text renders correctly.

<div data-no-scramble="true">
    <pre><code>This text will not be scrambled.</code></pre>
</div>

CLI Usage

The webshield.js script provides a command-line interface. It reads an HTML file, obfuscates it, and writes the output files to a directory.

# Multi mode (default, defeats frequency analysis):
rampart webshield.js page.html 42 output/

# Simple mode (1-to-1 substitution):
rampart webshield.js page.html 42 output/ --simple

# Reuse existing font + mappings:
rampart webshield.js page.html 42 output/ --reuse

# Enable image scrambling (requires rampart-gm):
rampart webshield.js page.html 42 output/ --images

# Custom tile size (default 32px):
rampart webshield.js page.html 42 output/ --images --tile-size 16

# Enable CDP guard (blocks Puppeteer/Playwright/Selenium):
rampart webshield.js page.html 42 output/ --guard

# Guard with custom delay (default 500ms):
rampart webshield.js page.html 42 output/ --guard --guard-delay 300

# Output:
#   output/page.html          - obfuscated HTML
#   output/fontname.ws.ttf    - scrambled font
#   output/imgname.ws.jpg     - scrambled images (with --images)
#   <source_dir>/fontname-mappings.ws.json  - saved mappings

Using with rampart-server

To obfuscate pages dynamically on each request, load the module in a rampart-server app. Generate the font and mappings once at startup, then use the saved mappings to encode each page on the fly. In multi mode, every response uses different codepoints for the same text.

var server = require("rampart-server");
var ws     = require("rampart-webshield");

// Generate font, images + mappings once at startup
var init = ws.fontshuffle("template.html", 98765, {
    mode: "multi",
    images: true             // requires rampart-gm
});

// Write font and scrambled images to the web root
for (var i = 0; i < init.fonts.length; i++)
    rampart.utils.fprintf("html/" + init.fonts[i].name, '%s', init.fonts[i].data);
for (var i = 0; i < init.images.length; i++)
    rampart.utils.fprintf("html/" + init.images[i].name, '%s', init.images[i].data);

// Save mappings for reuse
var savedMappings = init.mappings;

server.start({
    bind: ["0.0.0.0:8080"],
    map: {
        "/": function(req) {
            // On each request, encode with saved mappings
            // (no font/image regeneration, just text remapping)
            var result = ws.fontshuffle(
                "template.html", 0,
                {mappings: savedMappings}
            );
            return {html: result.text};
        }
    }
});

Since multi mode uses Math.random() for alias selection, each response is different even with the same mappings. Scrapers cannot correlate repeated requests to crack the substitution.

Image scrambling

Images can also be scrambled using tile shuffling. The image below has been tile-shuffled — right-click and save it to see the scrambled version. The browser unscrambles it on the fly using a small canvas script.

Simple vs Multi mode

In simple mode, each character maps to exactly one scrambled codepoint. This is fast and works well when a new seed is generated per request. In multi mode, each character maps to many codepoints, all pointing to the same glyph. The encoder randomly picks from these aliases, so identical words look different in the source each time. This defeats frequency analysis on static pages.

Multiple fonts

Pages can use multiple font families, including bold and italic variants, and mix LTR and RTL scripts. Each font is scrambled independently but they share a consistent mapping so the same encoded text works across all fonts on the page.

See the multi-font test page for a live example with six font files (Inter Regular, Bold, and Italic; Noto Sans Arabic Regular and Bold; Noto Sans) covering English, Farsi, and Greek.

What Puppeteer sees

When the guard option is enabled, a CDP (Chrome DevTools Protocol) trap detects automation frameworks like Puppeteer, Playwright, and Selenium. Email and phone protection

When the guard is enabled, mailto: and tel: links are obfuscated in the HTML source. The links work normally for human visitors — clicking them opens your email client or dialer. But scrapers harvesting the raw HTML cannot extract the addresses.
